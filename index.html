Basic syntax

A variable is exactly what it was in algebra: something in which to store data. In javascript, we declare and store variables like this:

var a = 7; // anything after 2 /'s (on the same line) is a comment and won't be executed.
// so you can literally type anything!! a;lk ajd!

The keyword var tells the browser to make a new variable called a. The = then tells the browser to set it to the value 7. Finally, the ; tells the browser that the line of code is over -- All statements should end in semicolons!!

Then how come it returns undefined? We'll get to that later. You can check what any variable is just by typing it:

a
// => 7

I'll use the // => to denote the output of a line of code -- no need to type it (or any other comments, for that matter) in.

Types

There are (many) different types of variables. We'll learn about 3 today. A type tells the engine what kind of data a variable holds. For example, try this:

a = 7; // We don't have to type 'var' again because the engine already knows a exists; var is for only the first time you declare a variable
var b = "7";
a + b;
// => What will it be?

You may be surprised to see it returned "77". That's because a is of type number and b is of type string, or a collection of letters (called chars in comp sci talk). When you add two strings, you just concatenate them, as you'd expect (so "java"+"script" == "javascript"). The engine treated a like a string so that the addition could make sense. How else would you add 7 to, say, "cat"?

To check a variable's type:

typeof a // => "number"
typeof b // => "string"

Functions

Functions are also exactly what they were in algebra: something that takes input and gives output, and if the input is the same, the output is too. We write functions like this:

To type a new line in the console, use shift + enter

function (a, b) {
return a + b;
}

This is correct syntax, but won't work in the console (something about an unexpected '('). This is just because the console doesn't like multiple line statements, and we'll fix it in a minute, but first let's analyze what we typed. function tells the browser that we're declaring a function. Everything enclosed in {} is a part of the function: they tell the engine where the function starts and ends. (a, b) defines the input -- in this function, we're taking two pieces of input, called a and b. Finally, the return tells the function what to output, in this case, the sum of a and b.

Making it work

To make the code work, type this:

var func = function (a, b) {
return a + b;
}

This is cool! We've just assigned the a function to the variable func. To call (i.e. use) the function we use a similar syntax:

func(6, 7); // => 13
func; // => function (a, b) { return a + b; }

Notice the difference between checking what the function is (the second line) and using it. Now, what do you think you'd get by typing:

typeof func;// ?
// How about this one:
alert("howdy soldier!");

alert is that pesky function that some sites use to show messages. Now you know how to piss people off with javascript!

Notice that javascript doesn't make you specify the types of input. That is, this will work too:

func("cat", "dog"); // Love that show.
alert(17);
alert(func);
alert(func("cat", "dog")); // Curveball! What will this do?

The last line is an example of a nested function. The engine evaluates the inner function, then uses its return value as the input to the outer function. Just like using f(g(x)) back in Algebra!

JS and the DOM

You may be thinking, this is all very cool, but what does this have to do with the web? Well, Javascript has a special syntax for interacting with objects on the web page. Open up the console while viewing this page and type

document.get

Chrome should autofill with a lot of options, like getElementById and getElementsByClassName. Pick one and see what it does -- at this point, it should be pretty self explanatory. For example:

document.getElementById("6");
// => All the code for this page

Now what will the following do? Try it!

document.getElementById(6).style.color = "blue";
document.getElementById(6).innerHTML = "javascript rocks";

To undo changes, all you have to do is refresh the page -- changes made with javascript don't persist, that is, they never change any of the original code that made a web page!

If the code isn't self-explanatory, don't worry: you will never again use it, because we have jQuery.

The above javascript is pretty cool, but it involves a lot of typing. And what if I didn't want to get #6 but something more complicated, like #footer a.some_class? Then things get tough.

Enter jQuery

jQuery makes all this easy. Instead of the annoying getElementById, we can just use CSS selectors to get elements. Try:

$("#7"); // The same thing as document.getElementById("7") -- so much less typing!
$("code").css("color", "blue");
$(".nav").html("javascript rocks"); // Check the next and previous buttons!

This is why we love jQuery, and it's essential that you know it as a web developer. No one (literally, no one) works in plain javascript anymore: everyone uses jQuery. Much, much more can be found at jQuery.com

I wanna do it myself!

I'd hope so. Unfortunately, jQuery doesn't come built-in with the browser like javascript does. If you look at this page's source, you'll notice this in the head:

<script type="text/javascript" src="http://ajax.googleapis.com/ajax/libs/jquery/1.7.1/jquery.min.js"></script>

All you need to do is copy that into your own pages, and you have jQuery! To write javascript files (you don't want to be typing into the console all the time), make that scripts folder in your project that we always talk about and never use, and create a main.js file in there.

A working example

Here is the example we did from class yesterday. First create a folder for the project (i.e. Day 5). In it, make an index.html. Make a stylesheets and scripts directory as usual. In the scripts folder, make a main.js.

// index.html
<!doctype html>
<html>
<head>
<!-- include jQuery -- >
<script type="text/javascript" src="http://ajax.googleapis.com/ajax/libs/jquery/1.7.1/jquery.min.js"></script>
<!-- THEN include your own script -->
<script src="scripts/main.js"></script>
<link rel="stylesheet" href="stylesheets/style.css">
</head>
<body>
<p id="test">hey</p>
</body>
</html>

Some very basic stuff. Notice that we include jQuery first, then our main.js script. Now for the kicker:

// scripts/main.js
$(document).ready(function() { // We'll explain this line later
   console.log($("#test"));
   $("p").animate({fontSize: "30px"}, 1000);
}); // always remember to close your {}'s and ()'s!
// We'll go over what the {fontSize..} means later as well.

And that's it! Open up index.html in a browser and watch that text grow (the 1000 is the number of milliseconds the animation will take). In JavaScript, time is always in milleseconds.

Now you can use jQuery, CSS and HTML -- you know everything this class has to teach. All that's left is review, learning more functions and practice!
